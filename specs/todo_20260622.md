### A.3 任务 (a) 设计 — Modality-C 真实评测
**判别目标**：真实版面上 `A-fail ∧ (B/C)-succeed ⇒ perception bottleneck`；`A-fail ∧ C-fail ⇒ capability`。这正是 §8 说做不了的实验。

**Tier-1（headline，纯计算、零人工、零偏倚）**——用 PPTX 源把"人工标注"绕掉：
1. Zenodo10K 采 **~150–300 真实 slide**（功效对齐现有表的 90/90 量级）。
2. python-pptx 抽 IR = oracle B；LibreOffice headless 渲 PPTX→PNG = 真实图像 A。
3. **渲染保真审计**（复用 `part3_p3_fidelity.py`）：真实 deck 非 snap-to-master，预期保真率显著高于合成模板 → 顺带在真实数据上验 §4 的"模板吸收 45%"hazard（正向结果）。
4. 程序化注入 taxonomy 缺陷（G1–G6 / S / **G7**）→ clean+defective 配对 → de-leaking view 去标记（复用 `part3_elicit.py` 的 `clean_variant`/marker-strip）。
5. 跑 modality **A / B / C × {detect, localize, repair}**，模型用 `part3_p1_roster.py` 的 6 模型（或省成 3 个 capable：Qwen3.5-27B + Gemma4-31B + InternVL3.5-8B）。
6. 指标沿用全文：paired-clean **balanced accuracy + Wilson CI + paired McNemar**，按类报；输出 A↔C 的归因表。

> "real" = 版面/内容分布真实；缺陷仍注入（与全文一致）。被堵的洞 = "真实几何 + 真实无损 oracle"，正是判别轴。

**Tier-2（可选，真·自然缺陷，要人工但防偏倚）**——仅当审稿人坚持"真实自然缺陷"时做：
- 纯图像无源文件子集（如 **SlideAudit**，已有第三方缺陷标签）。
- oracle = "自动检测器（layout+OCR）预填 + 人工纠错"（只改不从零标，省工时去工具偏向）。
- **防 self-annotation bias 硬措施**：指南**跑模型前冻结**；标注者**独立**（非作者）；对模型输出与"哪张缺陷"**双盲**；≥20% 双标报 **Cohen's κ / Krippendorff α**；**缺陷标签直接用 SlideAudit 第三方的**（不自标缺陷）→ §8 偏倚被堵死。
- 工时：~200 张 × 纠错式 4–6 min ÷ 2–3 人 ≈ 半天到一天。

### A.4 任务 (b) 设计 — reward 审计 ≥2
**协议沿用 DocReward 那套**（`part3_p3_reward_audit.py` 扩成多 RM）：
1. **同一批**配对 (clean, defective)，含 G7 的 90/90，逐类跑。
2. **统一 elicitation**（各 RM 输入契约不同）：pointwise 标量算 `Δ=s(clean)−s(defective)`；pairwise/BT 算 `P(clean≻defective)`；需要 prompt 的（PosterReward 要 design brief）给**固定中性 prompt**，并报一个 prompt 敏感性小检查。每 RM 协议写清。
3. **指标**：per-class `preference accuracy = P(reward(clean) > reward(defective))` + 95% CI（Wilson / pair-level bootstrap）；**G7 ≤ 0.5 = 盲/反向**。复刻 Figure 5（`part3_p2_figures.py`），多 RM 多序列。
4. **接回主线**：多个 trained RM（固定 elicitation ≈ C0）在 G7 一致失明，而重新 elicit 的 VLM-C3 抓得到 → 两边夹住论点，对称 6-VLM elicitation generality。
5. **泛化保真审计**（`part3_p3_fidelity.py`）：把 45% snap 后 identical 的配对喂**每个** RM → 预期 pref acc≈0 / gap≈0 → "任何 IR-标签 reward 数据都继承零信号配对"从一例扩成普遍。

**坑**：RM 输入格式异构（已给统一方案）；PosterReward 需 ms-swift（跑不通就记 deferred，不挡 ≥2 交付）；下权重前核 license + Ampere 可跑性。
