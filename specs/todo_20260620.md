## PART A — Spec & 实验协议（先读这里）

### A.0 一句话主张（thesis）
理想的幻灯片缺陷 critic 是 **符号-神经混合**：
- **细几何 / 规则类**（G3 对齐、G4 字号、G5 品牌色、G6 边距、细 G2、S3 术语、S4 密度）→ **符号 linter**
  （有坐标/规则，≈0.75–1.0、0 FP；DesignLab 自证神经模型即便喂坐标在 placement 上只有 0.149 recall）。
- **文本/结构语义**（S1 标题正文、S2 叙事顺序）→ **LLM**。
- **感知得到但被评测格式压掉的渲染类**（G1 溢出、S6 图文矛盾、**G7 渲染级越界**）→ **VLM**，
  但必须换 elicitation（**原子二元问 per-type + 强制证据** / 自由描述→归类 / 合成孪生→pairwise），
  而不是把整个 taxonomy 塞进一次 pointwise+rubric 绝对判定。

### A.1 已确立的事实（本计划的地基，勿重复验证）
1. 三模型（zs-8b / ft-8b / zs-30b）在 pointwise+rubric+modality A 下对一张真实"目录页越界"图全部 `has_defect=false`。
2. **同一张图、同一 30B，自由提问下准确点名"05 越界"** → 失败在**校准/格式层**，不是纯感知。
3. 细几何（G3 2–32px、tiny G2）即便换分辨率/编码器/尺度/forced-choice 仍在地板 → **亚感知，不可救，留 linter**。
4. linter 判**声明 bbox**，盲区 = **结构合法但渲染坏**（字形/图片越界、容器溢出）= **G7**，linter 和当前神经 reward model 都漏。
5. 探针基础设施可用：`playground/probe_toc.py` + `playground/serve_probe_toc.sh`；服务坑见 A.7。
6. **PresentBench（Tsinghua, arXiv 2603.07244）机制可借鉴**：holistic / 一次问全部 → judge 认知过载、含糊、易弃权；
   拆成**原子二元 checklist（每条单独一次调用）+ 强制定位证据**才可靠、且人类对齐（其 judge 单 item 单 call，~54 item/实例）。
   → 这是 C3 的动机来源与引用。注：它是**生成评测、内容为主、无 linter、不对几何判定测对错** → 不与本工作冲突，是 building block。
   其 Table 3/4 自陈 "Visual Design & Layout 分最低、跨模型差距最大，需 dedicated visual/rendering pipelines"——可引为"需混合 critic"的外部支撑。

### A.2 缺陷的"瓶颈二分"（决定每类走哪条路）
| 类型 | 瓶颈 | 归属引擎 | 本计划是否尝试用 VLM 救 |
|---|---|---|---|
| G1 溢出 / S6 图文 / **G7 渲染越界** | 校准/格式 | VLM | ✅ 是（Protocol 1 的核心）|

### A.3 G7 缺陷类定义（新增，需造数据）
**G7_RENDER_CONTAINMENT_OVERFLOW**：元素的**声明 bbox 合法**（在页边距内、bbox 之间无 IoU 重叠），
但**渲染后的内容越出其容器/卡片/页面边界**。判据：
- linter（基于 bbox）输出 `has_defect=false`（必须验证 linter 确实看不见 → 这是 G7 的定义性质）；
- 人/VLM 自由模式能看见越界。
合成方式（HTML 渲染器，freeform，**禁用 snap-to-master 以免吸收**）：
- (a) 卡片容器高度不足 → 末列表项渲染到卡片外（= 目录页案例）；
- (b) 不可断长串 / 超长文本，声明 bbox 在边距内但渲染文字超出卡片右沿；
- (c) 图片元素声明 bbox 合法但 `object-fit` 导致渲染内容溢出框。
每种 ~20–40 对（defective + 同底图 clean），共 ~60–100 对。

### A.3.1 数据与评测对齐协议（taxonomy 对齐 / OTHER 桶 / demo 样本）
1. **采用 SlideAudit 的 taxonomy 做对齐，别自造一套。**
   - 评测的**规范类别 = SlideAudit（UIST'25, arXiv 2508.03630）缺陷 taxonomy**；本项目 G1–G7 / S1–S6 一律**映射**到它，
     沿用并扩展 Part 2 已建的"SlideAudit 7 维 → G1–G6+S4"映射，落成 `data/part3/taxonomy_map.json`（双向）。
   - **G7 是本项目的扩展类**（SlideAudit 未单列"渲染级越界"）→ 在映射表里显式标注为 our-extension，并指向其最接近的 SlideAudit 类，
     报告里讲清"我们在其 taxonomy 上加了一类，并给出判据"。
   - 收益：结果**直接与 SlideAudit / PresentBench 可比**；避免自造 taxonomy 的标注偏见；省掉自标注大头。
2. **保留 `OTHER` 桶捞 off-taxonomy 缺陷。**
   - **C1（free-form→classify）的归类步必须有 `OTHER/uncategorized` 出口**：映射不到任何已定义类型的自由描述进 OTHER，并保留原文。
     （C3 是 per-type 二元、只覆盖已知类 → 另设一次"开放扫描"调用 "还有没有上面没问到的问题?" 来收集 off-taxonomy 项。）
   - **跨样本反复出现在 OTHER 的条目 = 候选新缺陷类型**（Phase 5 汇总）→ 这是 open→taxonomy **相对固定 checklist（PresentBench）的真差异点**：固定表漏掉的，前端能捞出来。
   - OTHER 不计入 precision/recall 的已定义类别评分，但单独统计其量级与复现率。
3. **小红书 / 手集真实图仅作 demo 样本，不进定量评测。**
   - 放 `data/part3/demo_real/`，明确 tag `non-eval`；用途 = 报告配图、定性案例、冷邮件钩子（如金字塔标题遮挡那张）。
   - 原因：无标注、选择偏置 → 不能算分。**定量真实数据评测一律走 SlideAudit（有标签）**。
   - 每张 demo 附一行人工说明（缺陷描述 + 映射到哪个 SlideAudit 类 / 或 OTHER），仅供定性展示。

### A.4 Protocol 1 — Elicitation 诱发恢复（#1，**load-bearing**）
**问题**：对"可救类"（G1/S6/G7），换 elicitation 能否把 pointwise 压掉的检测能力恢复出来，且 precision 可用？

**条件（每 模型 × 每缺陷类）**：
- **C0 baseline**：pointwise + rubric + modality A，**整个 taxonomy 一次调用**（现有 `part2_eval.py pointwise`）。预期 ≈ 弃权/0.50。
- **C3 atomic-binary（PresentBench 式，主对照）**：**一种缺陷一次调用**，问成二元
  "这张图有没有 <type X>?YES/NO + **必须给定位证据**（element/region）"。即把 C0 的过载单次调用拆成 N 次原子调用。
- **C1 free-form→classify**：两段式。Stage1 自由描述"这张图哪里不对"（高召回）；
  Stage2 用便宜 LLM 把描述映射到 taxonomy + 定位，输出 `{type, present(bool), locator}`。
- **C2 synth-twin pairwise**：用 `slide_examiner/template.py: snap_slide_to_master`（或几何归一化器）
  对**待测原图**生成"几何归一化反事实孪生" → VLM pairwise（双序消位置偏置）→ "原图更差"⇒缺陷存在。
- （可选 **C4** linter 提名区域 → VLM 原生分辨率裁剪复核。）

> **核心科学对照 = C3 vs C0**：同模型、同 taxonomy、同图，唯一差别 = "一次问全部" vs "原子拆分 + 逼证据"。
> 若 C3 ≫ C0，**直接证明 pointwise 弃权是认知过载/格式 artifact、非能力缺失**——这是"格式抑制"论点的主证据（PresentBench 背书）。
> elicitation 谱系预期：C0（过载弃权）< C1（自由，高召回高噪）≈ C3（原子，平衡，高 precision）≲ C2（专攻感知-未校准类）。

**模型**：~~zs-8b、zs-30b、ft-8b（3）~~ → **改为 model-agnostic 6 模型 / 4 家族**（2026-06-21，见 Phase 2）：Qwen3.5-9B/3.5-27B/3.6-27B + InternVL3.5-8B + Ovis2.5-9B + Gemma4-31B。
**指标**：paired-clean **balanced accuracy** + recall + FPR + **precision**（沿用 `part2_eval.py: score_pointwise`）；**C3 的强制证据已计入定位正确性**（`part3_p1_summary.py: localization_g7`，对 G7 GT 比对 region+element，capable 模型 ≥98%）。
**数据**：合成 = `manifest_eval_test_rendered`(G1/S6) + 新 G7 集；**用 freeform 变体**（避开 snap 污染）。
**判定（可证伪）**：
- 成功 = C1/C2/C3 任一的 balanced-acc 显著 > C0，**且 precision ≥ 0.7**（否则过报使混合不可用）。
- **C3 vs C0 单独成表**，作为"格式抑制非能力缺失"的直接证据。
- 失败 = 所有 elicitation 的 precision 都太低 或 自由模式也看不见 → 记为诚实负面（"G7 仍是开放盲区"）。
