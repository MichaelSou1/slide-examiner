## E1 — 2-AFC confound disentanglement  **(P0, gating)**

**Why.** R1-W2 + DA-C1: the central inference ("format suppression, not capability") is drawn from
forced-choice recovery (Fig 2: pointwise 0.50 → 2-AFC 1.00 on G1/S6) and C3 atomic-binary recovery on G7. But
2-AFC supplies (a) a guaranteed clean contrastive reference and (b) chance fixed at 0.5, and C3 supplies a
**named target** C0 withholds. The design conflates *elicitation* with *task difficulty / information
available*. We must factor the 0.50→1.00 gap into its components.

**Design — add two conditions** holding model + image fixed:

1. **`C0_named`** (new): single-slide, *absolute* (no clean reference, no pairing), but the prompt **names the
   candidate defect type** and asks an atomic yes/no — *without* C3's forced-evidence/localization gate.
   Isolates "naming the target" from "pairing" (vs 2-AFC) and from "forced evidence" (vs C3).
2. **`AFC_clean`** (new): forced-choice over **two clean slides** ("which has the {defect}?"). Measures the
   contrastive-guess floor + position bias that inflates the 1.00 — the always-pick-one artifact.

**Decomposition of the G1/S6 gap** (pointwise C0 0.50 → 2-AFC 1.00):
- `C0_named − C0` = the **naming/taxonomy-coverage** component (format suppression proper).
- `2-AFC − C0_named` = the **reference-availability/pairing** component (task difficulty).
- `AFC_clean` deviation from 0.5 = forced-choice **guess/position-bias** inflation.

**Code.**
- `scripts/part3_elicit.py`: add `engine_c0_named` (clone `engine_c0` @124; inject `target_defect` into the
  prompt as a named atomic yes/no; parse `has_defect` at detection level, **no** forced-evidence gate) and
  `engine_afc_clean` (reuse the pairwise plumbing from `slide_examiner/elicit_pairwise.py` but feed two clean
  renders; pre-render twins in the main thread — Playwright sync API is not thread-safe, per the C2 gotcha).
  Register both in `ENGINES` @205. Scoring: `C0_named` scores paired-clean exactly like C0 (the clean twin
  gets the same named yes/no → measures specificity, so balanced accuracy is well-defined and the "always-yes"
  artifact is controlled). `AFC_clean` reports pick-rate + position-bias only (no true positive).
- Reuse `scripts/part3_p1_roster.py` to sweep the **6-model / 4-family roster** (Qwen3.5-9B/27B,
  Qwen3.6-27B, Gemma4-31B, InternVL3.5-8B, Ovis2.5-9B) × `{C0, C0_named, C3, AFC_clean}` × `{G1, S6, G7}`,
  modality A, `--max-per-defect 60`. Add a `--condition` plumb-through if the roster hardcodes the C-set.
- Summary: extend `scripts/part3_p1_summary.py` to print the 3-way decomposition + paired McNemar
  (`C0_named` vs `C0`, `2-AFC` vs `C0_named`) with **Holm correction from E2**.

**Data.** `data/part3/manifest_g7_rendered.jsonl` (G7, 90 pairs) + the G1/S6 freeform records from
`data/part2/manifest_eval_test_rendered.jsonl` (filter `"__template" not in image_path` — template renders snap
away ~45% of geometry; freeform only, per the P2 gotcha). 108 G1 / 36 S6 positives + matched cleans.

**Serving.** 2-server parallel (TP=2 each: GPU0,1@8101 + GPU2,3@8102) shards the roster ≈2×. Gemma4 on
`vllm-latest`, no fp8 KV. Disable thinking: `PART3_CHAT_KWARGS='{"chat_template_kwargs":{"enable_thinking":false}}'`.

**Outputs.** `data/part3/p1_decomp__{model}_{cond}_{defect}.json` → new section in `reports/part3_hybrid.md`
("Elicitation decomposition") + a new **decomposition figure** (stacked: naming vs pairing vs guess-floor per
class) → `paper/figs/fig7_elicitation_decomp.png`.

**Paper integration.** §4 (Diagnosis) / §5.1 (Elicitation) get the decomposition; Limitations' 2-AFC caveat
(already added) becomes a *resolved* point (or a *confirmed* one, per the gate). Update Fig 2 caption.

**Acceptance.** Every cell carries n + Wilson CI; the McNemar decomposition is reported with Holm-corrected p;
the naming-vs-pairing split is stated numerically for G1/S6 and G7.

**Falsification branch (pre-registered).** If `2-AFC − C0_named` dominates (pairing carries the recovery), we

availability-of-reference effect," soften the title's "not the eyes" to the format-suppressed subset where
